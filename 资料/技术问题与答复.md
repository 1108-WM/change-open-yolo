## 数据集与实验规模

### 为什么不能只用 3 个场景说明 ScanNet200 有效？

三场景只能验证数据路径、投影、GPU、输出格式和明显失败模式，不能代表 ScanNet200 的场景、类别和几何分布。当前随机 `even48` 用于方向筛选；只有在 `even48` 有明确 AP 正信号后，才用独立随机 `even96/odd96` 确认泛化。

### 之前的 even48 是否随机？当前如何保证？

旧 even48 的来源不可确认，因此不再作为可比依据。当前 `output/scannet200/scene_splits/` 由 `tools/generate_random_scene_splits.py` 按 seed `20260718` 从 312 个可用场景可复现生成：`even48` 为随机 48 场景，`even96` 包含 even48，`odd96` 与 even96 不重叠。文件名只为兼容旧脚本，不表示按序号抽样。

### 是否需要重新下载 Any3DIS 或官方 ScanNet 数据？

Any3DIS 不需要重新下载数据；当前方法不使用其 GT、预提取特征、专用 superpoint 或数据布局。当前 `data/scannet200` 已有 312 个官方 val 场景名，且每个场景都有 RGB、深度、pose、内参、`_vh_clean_2.ply` 和预处理 `.npy`；抽查中 `.npy` 行数与 mesh 顶点数相等，因此足以继续 IBSp、SAM2 轨迹和回投。

但该结构检查不能证明历史下载来源。为论文可复现性，建议取得授权后的官方 ScanNet **v2** 文件，在独立目录下载少量现用场景逐文件比对后，再决定是否迁移整个 val 集；不应直接下载 1.3 TB 全量 release 或覆盖当前工作目录。

### 官方 ScanNet v2 应下载什么？

项目不需要 v1、`--task_data` 的训练模型，也不需要官方 2D 标注投影作为推理输入。官方脚本下载根链接是 `http://kaldir.vc.cit.tum.de/scannet/download-scannet.py`，默认即 v2。用于可复现当前数据和必要时重建 ScanNet200 预处理的最小原始文件是：

- `.sens`：官方压缩 RGB-D、位姿和内参；用 ScanNet `SensReader` 导出当前项目所需的 `color/`、`depth/`、`poses/`、`intrinsics.txt`。
- `_vh_clean_2.ply`：当前点云和评估顶点顺序的基础。
- `.txt`：包含场景元数据/对齐信息。
- `.aggregation.json` 与 `_vh_clean_2.0.010000.segs.json`：仅用于按官方 ScanNet200 脚本重建标签化 `.npy` 或离线评估，不得进入推理。
- `--label_map`：获取最新版 `scannetv2-labels.combined.tsv`；ScanNet200 官方说明要求使用新版映射。

官方脚本每次 `--id` 只下载一个场景，建议先对当前 smoke 场景 `scene0019_01`、`scene0084_01`、`scene0084_02` 下载上述类型到新目录，校验 mesh、帧和 pose 与本地数据一致。确认后如需官方来源覆盖全部评测，按当前 312 个 val 场景名单循环下载这些类型；不要使用无 `--type` 的全量下载。

当前审计状态：独立目录已取得最新版 label map，`scene0019_01`、`scene0084_01`、`scene0084_02` 的五类文件均已齐全；`scene0084_02.sens` 经断点下载后已按 `608020272` bytes 校验。`tools/audit_official_scannet_v2.py` 已完成三个场景的只读比对并全部通过：mesh SHA256 相同、`.npy` 的全部 xyz/RGB 与官方 mesh 逐值一致，`.sens` 的帧数和内参一致，首/中/末帧的深度逐像素一致、位姿在 `1e-5` 内一致。RGB 的小像素差来自 JPEG 重编码。报告在 `docs/diagnostics/official_scannet_v2_audit_3scenes.json`。

这证明当前 `data/scannet200` 在这三个样本场景与官方 v2 数据兼容，足以继续现有方法实验；但三场景审计不能证明全部 312 个场景的历史来源，因此不覆盖、不迁移当前数据，也不在论文中声称已完成全量来源验证。

若官方 `download-scannet.py` 在 `.sens` 下载时报告 `SSLZeroReturnError` / `TLS/SSL connection has been closed (EOF)`，这是官方站点或中间网络在 TLS 握手时主动断连，不是数据类型、路径或本地 Python 环境错误。该 URL 已确认可访问并支持 HTTP Range；可改用 `curl --continue-at - --retry-all-errors` 下载到独立 `.partial` 文件，按已下载字节恢复，校验文件大小后再重命名为 `.sens`。

## 超点与 IBSp

### 原始 `.npy` 的第 9 列是什么？为什么评估只替换它？

第 9 列是 ScanNet 几何过分割的 superpoint/segment id，不是第 9 个数据集样本。当前 `run_evaluation.py` 在 superpoint refinement 分支只读取第 9 列作为点到超点的分组。受控 B/C 实验只替换这一列，才能隔离“superpoint 划分”带来的影响；第 10/11 列是 GT 相关字段，绝不能作为推理输入。

### 管线是不是只使用第 9 列？

不是。RGB-D、相机位姿、mesh、YOLO-World、SAM mask 和 3D 候选都参与推理。第 9 列只是在超点级操作中承担分组索引的角色。

### IBSp 是什么？

IBSp 是二维实例边界感知的 superpoint：先基于 mesh 几何/法线建立三维图，再将多视角 2D 实例边界与真实深度、可见性回投到图边上；被多视角稳定判定为跨实例的边会被剪掉，随后重分割得到更贴合实例边界的超点。当前实现不使用 GT 2D mask。

### IBSp 是否必须先做 YOLO-World + SAM？会不会比直接生成超点麻烦？

纯几何 superpoint 可以在最开始直接生成，许多论文就是这样做。IBSp 的创新点恰恰是先收集 YOLO-World + SAM 的二维实例边界证据，再细化几何 superpoint；因此它比纯几何超点多一步，但获得了二维实例边界约束。当前路线是“高覆盖二维观测 -> IBSp -> SAM2”，而不是先用 YOLO/SAM 生成完整三维实例再生成超点。

### 参考论文的超点在什么时机生成？

- Any3DIS：先用几何/法线图切分得到预计算 superpoint，再由其选提示点、关键视角，SAM2 轨迹提升回这些 superpoint，并做 3D mask optimization。
- Details Matter：同样先有 3D superpoint；二维实例 mask 被提升为超点集合，随后通过 tracklet、共识、迭代合并和包含删除形成实例。
- 当前方法：保留“先有超点再以超点形成实例”的主体顺序，但在实例形成之前使用二维边界证据把纯几何超点替换为 IBSp。这是可研究的新位置，不是把二维实例结果直接当作超点。

### f30 是否表示“只用前 30 帧生成超点”？

不准确，必须拆成两个阶段理解。

1. **基础几何 superpoint**：对完整 ScanNet mesh 的全部顶点建立网格邻接与法线图，随后做图分割；它不是由前 30 个 RGB-D 帧从零生成，场景中所有 mesh 点都参与。
2. **f30 IBSp 边界细化**：从按固定间隔采样的 RGB-D 序列中取前 30 个实际采样帧，读取已有 YOLO-World + SAM 二维实例观测，把可见、深度一致的二维边界回投到上述几何图边。多帧稳定认为两侧属于不同二维实例的图边会被剪除，再重新得到第 9 列 IBSp id。f30 因而会影响 IBSp 的边界细化，但不是基础几何图的点集来源。
3. **SAM2 轨迹与种子资格**：同一 f30 观测范围也被用于 superpoint 可见性、关键视角、提示点和轨迹回投。当前“至少三帧可见”的资格门槛是在这一层起作用，不等于“超点没有生成”。

因此，“无可靠超点”首先应理解为“已有 IBSp 没有满足当前可见性资格”，而不能直接解释成几何 superpoint 划分失败。

### 哪些论文真正详细讨论 superpoint 生成？

建议分两类阅读，而不是只摘录“使用法线图割”。

- **几何超点的基础构造**：Landrieu 与 Simonovsky 的 *Large-scale Point Cloud Semantic Segmentation with Superpoint Graphs*（CVPR 2018）详细给出几何同质分区、图边、特征和图划分目标，适合补全“mesh/点云怎样先切成几何超点”的理论与实现依据。它不是当前二维边界感知创新的直接替代。
- **二维实例边界约束超点**：本地 `related papers/2026（ICLR）OVSeg3R Learn Open-vocabulary Instance Segmentation from 2D via 3D Reconstruction.pdf` 的 2D Instance Boundary-aware Superpoint 是当前 IBSp 最直接的论文依据：重点不在重新定义基础几何图，而在将可见、深度一致的二维实例边界转化为三维图的剪边约束，避免普通几何超点跨越二维实例边界。
- **多视图局部再细化**：本地 `related papers/2026（CVPR）.MV3DIS.pdf` 讨论以粗三维段为锚，按可见性、二维 mask 覆盖、深度一致和三维覆盖选择观测后，重新计算局部 superpoint 亲和；它适合研究“已有粗超点如何局部重分配”，但当前 oracle 不支持直接接入。

早期单项诊断显示，漏检对象内仍存在平均约 `91.5%` 归属单一实例的最佳 IBSp，而无可靠资格的直接原因是 f30 可见性与固定三帧门槛。该结论只说明“可靠性筛选”不是由超点过小引起。随后完成的原始/IBSp 全链路对照表明，当前 f30 IBSp 已在更早的重分割阶段系统性降低边界覆盖；因此现在的正确顺序是先做几何基线粒度对齐，再调资格策略。

### 原始 ScanNet superpoint 与当前 f30 IBSp 到底在哪一步开始变差？

新增 GT-only 工具 `tools/diagnose_superpoint_pipeline_gt.py`，结果在 `docs/diagnostics/superpoint_pipeline_even48_20260722/`。它定义“实例内主属超点覆盖”为：一个 GT 实例中，落入“该实例至少占该 superpoint 一半点”的 superpoint 内的点比例；这是超点边界质量的离线解释指标，不能进入推理。

结果显示下降发生在 f30 IBSp 重分割，而不是后续 SAM2：原始 ScanNet superpoint 对 1,113 个有效 GT 实例的平均该覆盖为 100%，当前 f30 IBSp 为 86.2%；对 85 个 Mask3D 漏检实例为 100% 降至 75.8%，74 个实例变差、11 个不变、没有实例变好。f30 之后才有可靠性筛选、最多三轮采样、SAM2 跟踪和提升，因此这些后续模块不可能修复已经被超点合并吞掉的边界。

原始第 9 列是 ScanNet 预计算的 mesh-normal `Segmentator` 分段；本项目已经实现 Python 版几何重分割，但 f30 IBSp 从完整 mesh 重建而非在原始 id 上直接剪分。当前 f30 使用 `merge_k=0.25`，而本地 ScanNet `Segmentator` README 标注默认阈值 `0.01`；原始 `.npy` 没有保存历史参数，不能把默认值当作已证实事实，但此差异与 f30 分段变少、最大分段变大同时出现，是当前最优先的实现风险。先做无 GT 的基线粒度对齐，之后才调可见性资格。

### SAM2 接收什么输入？能否采用 Details Matter 的前序流程？

当前 SAM2 **直接**接收的是：关键 RGB 帧、从可靠 f30 IBSp 投影到该帧的三个正点提示、以及按间隔采样的 30 帧 RGB 序列。superpoint 不是送入 SAM2 的像素 mask；它在前面用于选择种子与关键视角，在后面用于把 SAM2 的二维轨迹提升成三维实例。

Details Matter 原文不在 Grounded-SAM mask 后重新生成 superpoint。它使用预计算 superpoint，先将每帧二维 mask 按面积排序、从较大 mask 去除与较小 mask 的重叠，再用帧可见性比率和实例支持比率把二维 mask 提升/过滤到已有 superpoint，最后以帧级 sIoU 维护 tracklet、做多视图共识和迭代合并/删除。当前项目可以借鉴这一顺序，但不能直接替换 SAM2：应保留 SAM2 的跨帧传播，将“小 mask 优先去重”和“先提升再作帧级关联”作为独立受控模块。当前实现的同帧清理发生在 SAM2 传播后，且从所有重叠轨迹中同时删像素，尚不等价于原文。

### 原始 superpoint 对真实实例覆盖很好，为什么加入 SAM2 的 AP 仍然几乎不变？

“实例内主属超点覆盖”只说明一个真实物体没有在超点层被不可逆地吞并到另一个物体中；它不是完整实例预测，也不是 AP。原始 ScanNet superpoint 可以把一个真实椅子安全地切成许多碎块，仍需要二维候选或 Mask3D 给出“哪些碎块属于同一把椅子”、跨帧关联把观测合并、MVPDist 给类别、融合层给分数并与其他预测 NMS。任一环节错误，AP 都不会上升。

不加入 SAM2 的强基线流程是：Mask3D 类别无关 3D mask，加上 YOLO-World + SAM 形成的 `SAM-fused` 二维回投候选和 BPR 候选，使用 MVPDist 语义投票、superpoint 细化、分数排序与 NMS。历史统一 `run_evaluation.py` 入口中，原始 superpoint 的 AP 为 `26.4689%`，使用 f30 IBSp 且其余条件不变为 `26.6595%`；在相同入口追加旧版 SAM2 候选为 `26.6907%`，仅多 `0.0312` 个百分点。另一套只读重评测入口中，加入同帧歧义消解和独立重观测后的 SAM2 候选与强基线几乎持平。因此正确结论是：当前 SAM2 有过极小正信号，但没有稳定、可复现的总体 AP 提升，不能称为已经有效。

### 是否应“先过分割、再合并”？

是。这是当前最合理的方向：过分割保留边界，后续可以在 SAM2 轨迹、多视图证据和语义一致时谨慎合并；欠分割一旦把两个真实物体放进同一个 superpoint，后续以 superpoint 为原子单元的提升和跟踪几乎无法安全拆开。当前 f30 IBSp 的问题正是相反迹象：分段数下降、最大段变大，并且 GT-only 覆盖变差，说明它没有成为“更安全的过分割”，而出现了过合并。

下一版应以原始 ScanNet superpoint 作为安全的过分割底座：要么精确复现同粒度几何图后只剪二维冲突边，要么直接在原始段的内部按强多视图边界拆分；第一阶段不允许跨原始 superpoint 边界的无证据合并。真正的跨段合并留给后续 SAM2/Details Matter 的多视图实例形成，并始终保留 Mask3D 回退。

### IBSp 是否已经带来指标提升？

在强 B 基线上，原始 superpoint 的 AP 为 `0.264689`，f30 IBSp 为 `0.266595`，增量 `+0.001906`；但 AP50/AP25 分别轻微下降。因此它是小幅且混合的信号，不能单独作为有效改进或进入 even96/odd96 的依据。IBSp 仍可作为方法模块和消融项，但必须与 SAM2 轨迹后处理共同验证。

### 为什么当前 `0.261404` 低于历史强 B/C 的 `0.264689` 或 `0.266595`？

它们不属于同一个可直接比较的评测入口。历史 B/C 使用 `tools/run_scannet200_even48_ibsp_control.sh` 调用 `run_evaluation.py`，固定强 `SAM-fused + BPR` 条件后只替换 `.npy` 第 9 列 superpoint id；当前 `0.261404` 由 `evaluate_multiview_object_clip_correction.py` 的无语义修正入口产生，并写入空语义特征关闭类别校正。它是此前 Alpha-CLIP 受控实验的无修正基线，不是当前 SAM2 改动使 B/C 下降。

因此应比较同一入口下“强基线”和“强基线 + SAM2”的差值，不能以该绝对值同 `0.266595` 判断优劣。此前两者持平也不构成 SAM2 无效：738 个新增候选中仅 3 个通过了误继承的统一 `0.50` 分数门槛。现已增加按来源门槛，历史 `SAM-fused`、`BPR` 保持 `0.50`，SAM2 使用独立门槛，同时仍受重叠、superpoint 和候选预算约束；只需用已完成的 even48 输出重评测，不重跑 SAM2。

重评测结果为 `AP 0.261404 -> 0.261435`、`AP50 0.352537 -> 0.352678`、`AP25 0.404424 -> 0.404573`。因此实现错误已修复，当前组合也确有极小正向，但远不足以证明策略有效。主要原因是最终仍以 `.20` 语义分数保留附加候选：738 条中 229 条达到该门槛，按导出时已有实例重叠字段同时通过的只有 31 条，后续还会接受 superpoint 细化和去重；AP 的变化仅出现在 `monitor`。下一步应提高候选几何与语义的共同可靠性，而不是直接放宽门槛。

### 当前 SAM2 结果为何不能同 B/C 直接比较？

除评测分数口径不同外，审计还发现此前的 `evaluate_multiview_object_clip_correction.py` 缺少 `--processed_scene_root`。历史 B/C 的 `run_evaluation.py` 可在 C 中读取 f30 IBSp 的 `.npy`，而当前 SAM2 轨迹和 MVPDist 候选虽来自 f30 IBSp，最终融合时却固定读取原始 `data/scannet200` 的 superpoint。这是混合链路，不应被称为“f30 IBSp + SAM2”的完整对照。现已补参数并让重评测脚本默认使用 f30 IBSp；下一次只读现有候选的重评测才具有一致的 superpoint 口径。

需要做系统自查与 GT-only 测评，但 GT 只能用于离线诊断，不能作为推理、参数选择或候选分数输入。优先级是：先做接口契约审计，逐项核对场景列表、点索引、superpoint 根目录、候选来源、类别编号和分数语义；再做候选形成的几何诊断、MVPDist 类别诊断、融合保留/拒绝原因诊断和最终预测误差诊断。每一层输出同一候选 id 的可追踪表，才能区分“没生成”“几何错误”“类别错误”“被重复/阈值过滤”或“最终排序压低”。

该自查已执行。无 GT 接口审计在 even48 的 48/48 场景通过；完整 GT-only 候选账本显示，当前问题不是“没有好候选”，而是“好候选与已有强基线重叠，追加规则拒绝它们”。738 条候选中有 151 条 IoU>=.50，分数>=.20 的 229 条平均 IoU 为 `.463`；但低既有实例重叠条件保留的候选平均 IoU 仅 `.063`。因此下一步不应继续增加纯新增候选，也不能直接放宽重叠限制；应诊断哪些已有实例确实残缺，再研究只对高置信边界证据进行局部修正或保守替换。

在历史 B/C 统一入口、f30 IBSp 和统一评分下，当前冻结的 SAM2 候选带来 `AP 0.266595 -> 0.266907`、`AP50 0.346625 -> 0.347182`、`AP25 0.395200 -> 0.396167`。因此此前低于历史 C 的数值来自不一致的评测入口和混合 superpoint 根目录，不是方法把历史 C 降低。这个结果是有效但小幅的正信号；新加入的同帧二维 mask 重叠消除尚未参与完整 AP，且 even48 已被多轮诊断使用，仍需在新冻结版本和独立划分验证。

## SAM2、Any3DIS 与 Details Matter

### 当前是否已经接入 SAM2？

是，已接入官方 SAM2.1-small 的独立环境。`tools/export_any3dis_sam2_tracks.py` 使用 IBSp 超点可见性、邻域加权关键视角和三个正点提示，执行 SAM2 前后向轨迹；`tools/lift_sam2_tracks_to_superpoints.py` 将轨迹回投到 IBSp。

### 是否已经完整复现 Any3DIS？

没有，也不应直接运行其完整工程。当前实现复用了其关键思想：超点 FPS 选种子、邻域可见性选 pivot、SAM2 image-mask 初始化和双向跟踪、超过 memory window 的重现提示、逐帧超点候选动态优化，以及清理后未认领可靠超点的第二轮采样。尚未复现其完整大规模采样策略、所有数据/特征依赖和官方论文配置；当前三场景两轮运行只是无 GT 数据流与规则 smoke，不是论文复现结果。

### SAM2 与 Details Matter 是二选一吗？

不是。SAM2/Any3DIS 负责跨视角关联并产生类别无关轨迹；Details Matter 负责轨迹之后的 superpoint 共识、迭代合并、重复/包含实例删除与局部清理。当前不采用 Details Matter 的 sIoU 作为主关联器。

### “refined instances” 是什么？

它是离线产生的、类别无关的清理后三维实例候选：由一组 superpoint 和对应三维点索引组成，并带支持帧、分数或后续语义投票所需信息。它不是最终 ScanNet200 预测；只有补齐类别和置信度并通过融合，才能进入 AP 评估。

### 当前 SAM2 轨迹做到哪里了？

新版三场景 smoke 已完成两轮，每轮每场景最多 8 个 IBSp 种子。第一轮 24 条轨迹均以 image-mask 初始化，清理后保留 `6 + 7 + 5` 个实例；随后从未认领可靠超点取第二轮种子。两轮候选经连续 ID 合并后，`tools/postprocess_sam2_superpoint_instances.py` 将 `13 + 13 + 11` 个候选变为 `11 + 12 + 11` 个实例，发生三次跨轮合并，并在一个场景删除 23 个竞争超点。尚未做合并可视化质检，因而不能判断是否还需额外的空间连通域局部清理；也尚未做语义投票或 AP。

### 为什么不能立即跑 even48 AP？

当前轨迹仍只有类别无关实例，尚未补 Open-YOLO/MVPDist 语义与融合格式；每场景 8 个种子也过稀，不能代表候选召回。应先完成合并可视化质检和语义投票，再以更合理的种子数量在相同 strong B 基线上做受控 even48 AP。

## 当前论文路线

### 当前最终选择的组合是什么？

`YOLO-World + SAM 高覆盖观测 -> IBSp -> Any3DIS 式 SAM2 轨迹 -> Details Matter 式后处理 -> Open-YOLO 3D MVPDist 语义投票 -> 可选 OV3D-CG 低置信修正`。

PoVo 暂不加入第一版，以免任务扩展为自动词表发现、论文主线过宽。OV3D-CG 也只在候选质量稳定后用于低置信、语义冲突和长尾类别。

### 当前可以接入 Alpha-CLIP 测 mAP 吗？

可以，且已完成当前随机 `even48` 的有效受控测试。重新导出的 Alpha-CLIP 缓存与 `even48.txt` 为 48/48 完全一致；在相同候选、相同阈值和同一评测入口下，无语义修正为 `AP 0.261404 / AP50 0.352537 / AP25 0.404424`，启用保守 Alpha-CLIP 类别修正后为 `AP 0.263244 / AP50 0.354621 / AP25 0.406099`，即 `+0.001840 / +0.002084 / +0.001675`。共在 42 个场景中替换 2182 个候选类别。

这证明 Alpha-CLIP 对当前候选的类别判定有小幅正向帮助，但不是 SAM2、IBSp 或 Details Matter 的实例候选提升证据。该受控入口的绝对 AP 也不能与强 B 的独立运行直接并列；应以它作为固定的可选语义校正规则，随后在强 B 与强 B + SAM2/Details Matter 上分别做同条件消融。

### 为什么 Alpha-CLIP 只带来很小的 AP 增益？

实例分割 AP 首先受三维 mask 的 IoU、重复实例和置信度排序约束；只修正类别不能把低 IoU、漏检或重复候选变为正确实例。当前 Alpha-CLIP 在 42 个场景中改了 2182 个候选类别，但修正几乎全部集中在低分候选：原始分数中位数为 `0.0112`，2062 个低于等于 `0.20`，全部低于等于 `0.55`。这类候选中许多本就不能在 AP 匹配中成为高排序真阳性，因此可提升的上限有限；其余语义改正也会与错误改正相互抵消。

其他论文报告数个点的提升，通常使用更干净/更接近 GT 的二维区域、不同候选生成器与排名融合，或直接比较 Alpha-CLIP 和普通 CLIP 的密集语义协议，不能直接与本项目“固定 3D 候选后只替换低分标签”的 AP 消融比较。当前结果更说明候选实例质量、重复控制和 3D 几何覆盖仍是主瓶颈；应先评估 SAM2 + Details Matter 是否提高候选质量，再用相同 Alpha-CLIP 规则判断二者是否叠加。

### 能否用 GT 检查 SAM2 输出到底哪里有问题？

可以，但只能作为离线诊断和评估，绝不能输入到种子选择、SAM2 提示、轨迹合并、类别投票、阈值决策或最终推理。`tools/diagnose_sam2_refined_instances_gt.py` 强制要求 `--allow_gt_diagnostics`，并将 GT 使用范围写入报告。

当前三场景两轮 SAM2 + Details Matter 诊断显示：34 个候选只覆盖 79 个有效 GT 实例中的 `15.19%`（IoU>=0.25）和 `10.13%`（IoU>=0.50）。清理前为 37 个候选，两个 recall 相同，因此不是 Details Matter 删除了正确对象；主因是每场景仅 16 个种子造成覆盖不足，同时存在“高精度碎片”和“跨实例/背景扩张”两种边界错误。应先提高 GT-free 的 baseline-novel seed 覆盖并加入边界清理，不能直接跑 even48 宣称 SAM2 有效。

### 增加 baseline-novel SAM2 种子后，为什么 AP 仍没有提升？

第三轮把强 B 已覆盖区域排除，并提高种子预算后，候选几何覆盖确实提升到 `25.32% / 16.46%`（IoU>=.25/.50），说明候选生成方向有效。但在相同三场景、新 2D 缓存和相同阈值下，融合 AP 与基线完全持平。这不是评测器用临时分数提前过滤候选造成的，接口已修复为附加候选按最终语义分数过滤。

GT-only 语义诊断显示，几何 IoU>=.25/.50 的候选中，YOLO 临时外接框标签的精确率仅 `22.22% / 18.18%`，Alpha-CLIP 的 top-1 为 `0% / 0%`，且概率约 `0.07`。新增的 Open-YOLO 3D 原生 MVPDist 投票将该准确率提升到 `33.33% / 38.46%`，证明它应作为主语义头；三场景 AP 对照仍仅有 AP25 `+0.002199`，AP/AP50 持平，说明还必须对轨迹泄漏、碎片和背景候选做 GT-free 质量门控。Alpha-CLIP 只处理 MVPDist 低置信或冲突实例，不能作为唯一分类器。

### 当前 SAM2 + Details Matter 实现是否已经完整实现了论文思想？

不是完整复现，也没有发现能单独解释结果的显性运行、投影或评测接口错误。当前实现已具备：IBSp 种子和关键视角、SAM2 image-mask 初始化与双向传播、长不可见间隔后的重现提示、逐帧超点回投/Any3DIS 式贪心优化、可见性共识、跨轮合并、包含删除、竞争超点清理和 GT-free 结构门控。

但它是有意裁剪的组合实现，仍缺少两个与当前失败模式直接相关的 Details Matter 环节：第一，原文在提升到三维前会消除同一帧多个二维实例 mask 的重叠区域；当前各 SAM2 种子独立传播，直到三维 superpoint 阶段才处理竞争，无法在二维层把跨实例 mask 拆开。第二，原文以每帧独立二维观测和逐帧集合交并比形成或校正轨迹；当前主要依赖单个关键帧初始化后的 SAM2 连续传播，缺少跨帧独立重新观测对漂移的纠正。当前只用正点、默认单个初始 mask、30 个稀疏采样帧也会放大扩张和碎片风险。

因此，现有小增益不能解释为“SAM2 或 Details Matter 思想无效”；更准确的结论是：当前组合链路已跑通且无明显接口故障，但尚未实现最可能抑制混合候选的二维重叠区域消除与独立重观测确认，候选几何和语义排序仍不足以转化为 AP。

### SAM2 是否真的补出了 Mask3D 漏检实例，是否应改用 Details Matter？

不能把当前小 AP 增益解释成“SAM2 没有补充能力”。新增的严格 GT-only 互补性诊断 `tools/diagnose_baseline_sam2_complementarity_gt.py` 在 1113 个 `even48` 有效 GT 实例上逐实例比较基础 Mask3D 与当前 SAM2 MVPDist 候选，报告位于 `docs/diagnostics/baseline_sam2_complementarity_even48_20260721/`。IoU>=.25 时有 `19` 个“Mask3D 漏、SAM2 覆盖”实例，IoU>=.50 时有 `16` 个；另有 `29/21` 个实例中 SAM2 的 IoU 至少高出 Mask3D `.10`。这证明 SAM2 能补出一小部分漏检或边界较差的对象。

但该能力目前很有限：IoU>=.25/.50 时 SAM2 真正补充的比例仅 `1.71% / 1.44%`，而 Mask3D 覆盖、SAM2 未覆盖仍为 `811 / 802` 个实例；同时二者共同覆盖 `217 / 135` 个实例。后者中的许多 SAM2 候选正是几何较好却与已有实例重叠的候选，当前只追加低重叠候选的融合策略会拒绝它们。因此问题既有前端候选覆盖不足，也有后端只能追加、不能保守修正已有实例的路由限制。该测评尚未比较完整 `SAM-fused + BPR` 强基线，不能用它声称 SAM2 已补出完整强基线的漏检。

不应以 Details Matter 替换 SAM2。Details Matter 的关键并不是用列表代替 SAM2，而是利用逐帧独立二维实例观测、同帧重叠消除和跨帧一致性来形成或校正轨迹；SAM2 的强项是依据外观连续传播类别无关 mask。合理的下一版是 `SAM2` 负责提出跨帧轨迹，Details Matter 式规则负责在提升到三维前消除同帧歧义、以独立重观测确认或降权传播结果；随后仅对高置信且确实改善局部边界的重叠候选研究局部修正/替换。不能以全面放宽重叠限制或直接换掉 SAM2 代替该验证。

### 低互补率是否因为超点不够、无约束二元优化和迭代采样没有实现？

不能只靠直觉判断。新增 `tools/diagnose_sam2_seed_coverage_gt.py` 后，IoU>=.25 的 85 个基础 Mask3D 漏检实例被定位为：`38` 个无可靠超点、`6` 个有可靠超点但未采样、`22` 个已采样但轨迹或三维提升未恢复、`19` 个被 SAM2 覆盖；IoU>=.50 的 176 个漏检实例对应 `65/31/64/16`。报告在 `docs/diagnostics/sam2_seed_coverage_even48_20260721/`。这里的“无可靠超点”是指该 GT 内没有同时满足当前最小点数、至少三帧可见等无 GT 可靠性条件的 IBSp，并不等于场景中完全没有超点；它可能来自超点跨实例、超点过小、遮挡或可见性过滤过严。

所以答案是：超点与可见性供给确实是首要问题之一，但不是唯一原因；当前 IoU>=.25 的直接采样预算缺口仅为 `6/85`，盲目增加轨迹不能解决 38 个没有可用种子的实例，也不能自动修复 22 个已采样后的失败。IoU>=.50 的未采样 `31/176` 则说明在边界质量目标上，增大预算仍值得作为单独受控消融。

无约束二元优化已经实现了近似求解：`lift_sam2_tracks_to_superpoints.py` 的 `any3dis_dp` 会逐帧比较“保留当前超点集合”和“加入本帧全部候选超点”，以所有帧投影中 mask 内点数减去 mask 外点数为目标，正对应 Any3DIS 论文的贪心动态方案。它不是逐字复现，因为候选超点先经过逐帧覆盖率筛选，优化后又施加了 Details Matter 式共识过滤。迭代三维物体采样也已实现，但只运行三轮：每场景最多 `8 + 8 + 16` 个种子，第二轮基于未认领超点，第三轮再排除基础 Mask3D 已覆盖区域；原文则继续到没有空闲超点。因此“完全没有实现”不成立，但“实现受到固定轮数、种子预算和额外过滤限制，尚未达到原文完整迭代范围”成立。

### 为什么漏检实例没有可靠超点，独立重观测如何实现？

完整 even48 的 GT-only 原因诊断显示，IoU>=.25/.50 下没有可靠 IBSp 的 `38/65` 个 Mask3D 漏检实例全部是“可见帧不足”，不是超点小于当前最小规模；其最佳超点的 GT 纯度均值约为 `91.5%/91.0%`。这仍支持后续研究 30 个 f30 采样帧和“至少三帧可见”是否过严；但原始/IBSp 覆盖对照已显示当前 f30 重分割先造成边界覆盖下降，故不能再据此否定 IBSp 实现问题。先对齐基础几何分割，再做可见性自适应。

独立重观测已经实现：每隔 5 个采样帧，在 seed 可见处用独立 SAM2 图像预测器生成 mask，与视频传播 mask 计算一致 IoU；低于 `.30` 时不修改原始轨迹，只在三维提升的超点支持、全帧优化和共识中降低该帧权重为 `.50`。24 条三场景轨迹得到 68 次检查，51 次通过、17 次否决，平均一致 IoU `.637`。硬删除否决帧会伤害 IoU>=.50 召回，软降权保持本次 `.0794/.0635` 的 IoU>=.25/.50 离线召回，因此只作为下一冻结版本的保守约束，尚不能声称提升。

### 三场景后是否应运行 even48？

应当运行一次。三场景只足以发现数据流、投影、输出和典型失败模式，不能推断总体 AP。下一次 even48 应采用当前冻结的 SAM2 三轮、Details Matter 后处理、GT-free 质量门控和 MVPDist 语义，和同一强 B 做单一受控对照；它用于验证当前版本的总体方向，不用于在 48 场景上反复调阈值。完整任务运行期间只检查启动、中段/异常和完成三个状态，避免高频轮询。

已检查现有输出：没有该完整配置的 even48 结果。已有 even48 的 IBSp 原型轨迹、SAM 融合或其他候选图实验虽然也使用了部分相近模块，但候选形成、后处理和语义归属都不同，不能拿来替代本次比较或重新解释为当前方法的结果。

### SAS 是否提出了 MLLM、CLIP、Alpha-CLIP 的实例选择器？

不是。SAS 的核心是：先将多个二维模型的特征对齐到同一 CLIP 空间，再用合成类别图像离线估计“各模型擅长识别哪些类别”，并据此融合模型特征、蒸馏三维编码器。它没有按每个类别无关实例在 MLLM、CLIP、Alpha-CLIP 之间作三选一。

可以借鉴为本项目的候选级软选择：用不含 GT 的几何质量、多视角一致性、MVPDist 置信度和类别间隔、Alpha-CLIP 置信度和类别间隔、以及两者是否冲突来决定语义证据权重。高置信一致候选直接使用 MVPDist；只有 MVPDist 不确定但二维对象证据干净时才允许 Alpha-CLIP 校正；只有几何质量高、两个模型冲突且属于难判类别的极少数候选才考虑 MLLM。先实现不调用 MLLM 的记录和加权接口；只有证明 Alpha-CLIP 对当前 SAM2 候选有条件正向作用后，才接入 MLLM，避免成本、幻觉和错误类别被放大。

### 当前不加 SAM2 的完整流程是什么，和加入 SAM2 后相比谁更好？

当前强基线先汇集三类候选：`Mask3D` 类别无关三维 mask、`YOLO-World + SAM` 的二维反投影候选、以及 BPR 候选；随后由 MVPDist 做开放词汇语义投票，再进行 superpoint 精炼、评分与 NMS。SAM2 不是替换其中某一项，而是在同一融合入口中追加一批由“IBSp 种子和关键帧 -> SAM2 二维轨迹 -> superpoint 三维提升 -> 后处理 -> MVPDist”的候选。

同一历史主入口中，f30 基线 AP 为 26.6595%，追加现有 SAM2 候选后为 26.6907%，只高 0.0312 个百分点；另一个 f30 一致的只读融合评测中，AP 基本不变。因而当前不能说“加入 SAM2 后整体更好”，更准确的是：SAM2 确实补到少量 Mask3D 漏检实例，但尚没有形成稳定、可复现的总体 AP 提升。相对真值的超点边界安全性则是原始 ScanNet superpoint 更好，原始为 100%，当前 f30 IBSp 为 86.2%；但这不是完整推理 AP 的直接比较，不能据此宣称原始超点的最终系统必然更高。
