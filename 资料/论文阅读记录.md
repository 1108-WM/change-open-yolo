# 论文阅读记录：当前路线的依据

本文件只保留与“二维实例边界感知 superpoint + 跨视角实例形成”直接相关的论文结论。具体实验状态见 `CURRENT_EXPERIMENT_STATUS.md`，当前方法边界见 `资料/当前基线修改方向.md`。

## 当前任务的关键难点

开放词汇 3D 实例分割通常同时受三类错误限制：二维 mask 覆盖不足或边界错位、三维候选跨物体或碎裂、多视角实例重复或语义不稳定。当前路线先解决前两类，再保留快速语义头处理类别。

## 与当前路线直接相关的论文

### Superpoint Graph

价值：本地 `Landrieu_Large-Scale_Point_Cloud_CVPR_2018_paper.pdf` 不只是把 superpoint 当黑盒。它先在十近邻图上以局部几何特征定义全局分区能量，再用 cut-pursuit 近似求解，使每个 superpoint 在几何上尽量同质；随后才建立 superpoint 邻接图。论文还明确指出，若初始分区把白板与墙等几何不显著对象合并，后续网络无法完全弥补这一上界。

对当前的启示：应把“几何基线分割是否与原始 ScanNet superpoint 同粒度”和“二维边界是否只删除有证据的跨实例边”分开验证。当前 f30 IBSp 的问题不是缺少 Felzenszwalb 算法，而是重新分割参数和原始基线不等价，导致二维剪边尚未带来净边界收益。

### Open-YOLO 3D

价值：YOLO-World 的开放词汇检测与 MVPDist 的快速多视角语义投票，适合作为效率较高的语义基线。

采用：保留其语义侧，不把主要创新放在替换检测器或语义分类器。

不足：其语义投票本身不能解决 3D 实例边界、重复候选和跨视角形成问题。

### OVSeg3R

价值：把多视角二维实例 mask 与真实深度投到三维图边上，在稳定跨实例边界处剪边，再生成更符合实例边界的 superpoint。

采用：当前 IBSp 的直接方法依据。先有二维观测，后剪三维图边；不是用 GT 二维标签。原文的关键顺序是：在基础 KNN 图中，若同一图边两端投影到二维实例 mask 后不属于同一实例，就不保留该边；随后才以法线边权运行 Felzenszwalb 分割。它不主张先用一个更粗的新分割替代已有几何基线。

风险：若二维 mask 稀疏或帧号/投影错位，剪边极少或产生伪边界。因此先检查覆盖、观测图边与背景污染。

### MV3DIS: Multi-View Mask Matching via 3D Guides

价值：不依赖视频轨迹，而以粗三维段投影为共同参考，从多帧二维 mask 中选择高可见、高 mask 覆盖、深度一致且三维覆盖分布相似的观测；用筛出的二维图重新计算 superpoint 亲和并迭代调整边界归属。

待验证适配：当前项目可在离线诊断证明二维证据可预测修正收益后，以原始 `Mask3D` 实例作为粗锚点，匹配 SAM2 轨迹中的可靠二维观测，只生成局部边界修正变体并保留原 mask 回退。精细几何后再做最终 MVPDist，粗候选的语义分布只作为匹配和合并的软证据；在该前提未验证前不得接入主流程。

证据：论文在 ScanNetV2 的消融中，粗分割 `33.3 mAP`，加入区域精细化为 `35.5`，再加入三维引导多视图 mask 匹配为 `37.0`，最后加入深度一致权重为 `38.5`。这支持“先稳定锚定、再用多视图二维证据局部修正”的顺序，而不是无条件用二维轨迹覆盖三维实例。

### Details Matter for Indoor OV-3DIS

价值：把 superpoint 作为三维实例的基本单元；其原文以 Grounded-SAM 观测提升到已有 superpoint，用 co-visible superpoint 的 sIoU 形成轨迹，再以多视角支持筛选、迭代合并候选、删除重复/包含实例，并做局部 mask 清理。

采用：SAM2 轨迹之后的 superpoint 可见性共识、迭代合并/细化、重复删除、包含实例删除和竞争单元保守清理。其帧级 sIoU 轨迹列表可作为 SAM2 观测的三维关联/接受器，不应简单替换 SAM2 视频传播。

关键消融：在论文的 ScanNet200 子集 class-agnostic 实验中，帧级 sIoU 相比轨迹级 sIoU 为 `AP +0.4`、`AP50 +1.8`、`AP25 +0.9`；较大增益来自完整链路：聚合后加入迭代合并/删除使 `AP25 +5.0`，再加入二维重叠处理与迭代多视图细化后，聚合-only 的 `AP/AP50/AP25=31.4/49.9/63.5` 提升到 `35.1/56.1/70.5`。因此论文高指标不能归因于“轨迹跟踪”单一模块，也不能与本项目不同评测入口的绝对值直接比较。

不直接照搬：不在第一阶段加入其完整分类体系；当前仍使用 Open-YOLO 3D 的快速语义投票。

### SAS: Segment Any 3D Scene with Integrated 2D Priors

价值：SAS 不是按实例把 MLLM、CLIP、Alpha-CLIP 三选一的选择器。它先把多个二维模型（论文以 LSeg 与 SEEM 为例）的输出通过文本对齐到共享 CLIP 空间，再以 Stable Diffusion 合成类别图像和 SAM mask 离线估计各模型的类别级能力，按该能力融合二维特征并蒸馏三维编码器。

可迁移思想：对固定候选的多语义证据，不应只按原始分数硬替换；可构造不使用 ScanNet GT 的类别/模型可靠度先验，对同一候选的 MVPDist、Alpha-CLIP 和后续上下文模型做校准融合或保守拒绝。

候选级软选择策略：先为每个类别无关候选记录几何质量、多视角一致性、MVPDist 的置信度与类别间隔、Alpha-CLIP 的置信度与类别间隔，以及候选内语义冲突。高置信且多视角一致时采用 MVPDist；MVPDist 不确定但对象 mask 干净、视角证据稳定时允许 Alpha-CLIP 参与校正；只有高几何质量但两个模型明显冲突、且属于长尾或难判类别的少数候选才提交给 MLLM。该策略输出加权后的类别和分数，也允许保守拒绝，不做强制三选一。

当前限制：SAS 的原始路线需要额外二维模型、特征空间对齐和蒸馏训练，与本项目的无训练候选追加链路不直接兼容。第一版只准备上述轻量候选级证据融合：先在不调用 MLLM 的情况下验证 MVPDist 与 Alpha-CLIP 的条件有效性和分数排序；未证明 Alpha-CLIP 对当前 SAM2 候选有效前，不启用 MLLM。

### Any3DIS

价值：先以几何 superpoint 选可靠单元和关键视角，再从 3D 点提示 SAM2，前后向跟踪类别无关 2D mask；轨迹回投到 superpoint，并以跨视角 mask 一致性优化实例。它给出了强的跨视角关联器。

采用：以 IBSp 替代其纯几何 superpoint，保留“可靠单元/关键视角 -> SAM2 跟踪 -> 轨迹提升到 superpoint”的顺序。SAM2 是最终跨视角关联器；现有共享核心匹配仅用于验证数据接口与 IBSp 原型。

### OpenMask3D

价值：确立“类别无关 3D 候选 + 多视角开放词汇语义识别”的基本分工。

采用：坚持实例形成与语义识别分层；先提高几何候选，再做开放词汇类别投票。

### OV3D-CG

价值：上下文推理能修正长尾、功能描述或低置信语义。

采用：只作为后期按触发条件调用的修正器，不进入第一阶段主干。

### PoVo

价值：自动发现可用词表，减少人工类别表依赖。

当前不采用：会把论文从“开放词汇实例形成”扩展为“无预设词表发现”，主线过宽。

## 统一方法判断

最有论文价值的组合不是把上述方法全部串联，而是：

```text
Open-YOLO 3D 的快速语义
+ OVSeg3R 式二维边界感知 IBSp
+ Any3DIS 式 SAM2 跨视角实例轨迹
+ Details Matter 式轨迹后 superpoint 共识、合并和去重
```

核心实验必须证明：更高覆盖的非 GT 二维观测使 IBSp 产生更可信的结构变化；再证明 SAM2 轨迹经过 superpoint 共识后比原始二维观测更少重复、更稳定。

## 不作为当前主线的方向

- 直接用 Alpha-CLIP 或固定阈值重分类：只能改类别，不能修复实例几何。
- 纯 kNN 超点替换：缺少二维实例边界感知，且不是当前创新点。
- 直接使用官方二维或三维 GT：只能做 oracle，不能进入推理或 AP。
- 一开始就接入 PoVo 或大模型上下文：会把候选形成问题扩展得过宽。
